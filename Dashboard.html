❓ PlanForge AI — Help Guide

Welcome to PlanForge AI — an intelligent project planning tool that converts BRD / SOW documents into complete Jira-ready project plans. Click any section below to learn how it works.

📤
Upload & Generate ⌄
The starting point. Upload your project documents and the AI generates a complete plan in seconds.
1
Choose Mode — BRD Mode (single .docx) or SOW + ARB Mode (two PDFs).
2
Enter API Keys — Paste your Anthropic API keys in the API panel. Keys are saved securely.
3
Set Dates — Choose your project start and end dates to set sprint cadence.
4
Upload Document — Drag and drop your file or click to browse.
5
Generate — Click the generate button. AI processes your document through 5 steps: Extract → Plan → Tests → Save → Ready.
💡 Tip: You can reload any previously saved plan using the "Reload Existing Plan" dropdown at the top.
📊
Live Dashboard ⌄
A real-time analytics hub with 6 tabs — Overview, Sprints, RAID, Tests, Health Score, and Charts. Auto-populates after plan generation.
→
Overview — Key metrics (Epics, Stories, Story Points, Sprints, Tests, Risks) plus a health heatmap.
→
Health Score — 0–100 composite score based on test coverage, story count, and risk density.
→
Charts — Story points per Epic, Sprint Velocity Trend, Story Complexity distribution.
📅
Gantt Chart + WBS ⌄
An interactive Gantt timeline with three views. A red line marks today's date across the timeline.
→
By Sprint — Each sprint shown as a colored bar across the project timeline.
→
By Epic — Each epic mapped to its sprint date range.
→
Full WBS — Complete Work Breakdown Structure: Epic → Stories → Subtasks with story points.
💡 Click any Gantt bar to see details. The WBS section below also shows the full hierarchy.
👥
Resource Loading Planner ⌄
Model your team composition and instantly see how changes impact the project timeline. This is where resource scenarios come alive.
1
Configure Resources — Add roles (Data Engineer, BA, QA, etc.) with FTE allocation, hours/day, and sprint range. The FTE slider per row lets you fine-tune allocation.
2
FTE Scenario Slider — Drag the slider from 0 to +5.0 FTE (in 0.5 increments) to explore what happens if you add resources. Use quick-select buttons for common scenarios.
3
Scenario Comparison Table — Compares all scenarios against the As-Is baseline. The "vs As-Is" column shows weeks saved, sprint count change, capacity gain, and FTE added.
4
Charts — Bar charts show Weeks to Complete and Sprint Capacity across all scenarios visually.
5
Timeline Bars — Proportional bars showing relative delivery speed per scenario (green = faster, amber = slower).
💡 Click any row in the table to activate that scenario. The summary cards at the top update instantly to reflect the selected scenario's metrics.
⚙️
Capacity Planner ⌄
Sprint-level capacity analysis. Tells you if your team can actually deliver what's planned each sprint.
→
Team Size — Number of developers/engineers on the team.
→
Hours/Day per Dev — Focused coding hours (typically 6 out of 8).
→
SP per Dev/Sprint — Story points each developer can deliver per sprint.
→
Buffer % — Overhead percentage for meetings, reviews, and unplanned work.
💡 Sprints showing ⚠️ Over capacity mean your team is overloaded — reduce sprint scope or increase team size.
🚀
Velocity Tracker & Smart Estimator ⌄
Tracks sprint velocity trends and provides a risk-buffered completion estimate.
→
Velocity Chart — Bar + line chart showing story points per sprint with average trend line.
→
Smart Estimator — Calculates completion date using velocity data plus a risk buffer based on RAID log severity.
🎯
AI Risk Radar ⌄
A spider/radar chart showing six dimensions of project health risk, each scored 0–10.
→
Risks — Number of RAID risks. Scores above 3 are flagged red.
→
Dependencies — External/internal dependencies. Higher count = higher exposure.
→
Complexity — Derived from total story points (every 10 SP = 1 complexity unit).
→
Test Cover — Test cases per story ratio. Higher is better.
💡 A well-shaped radar (scores near 0 on Risks/Issues, high on Test Cover) means a healthy project profile.
🗂️
Work Breakdown Structure (WBS) ⌄
A hierarchical tree showing the complete project decomposition: Epics → Stories → Subtasks + Acceptance Criteria.
→
Each Epic is collapsible and color-coded. Shows total story point count.
→
Each Story shows its story point badge, subtasks, and up to 2 acceptance criteria.
💡 Click the Epic header to expand/collapse its stories.
✅
Review & Approval → Jira Push ⌄
A two-level approval workflow before the plan is pushed to Jira. Both L1 (PM) and L2 (Solution Architect) must approve.
1
Both reviewers set status to ✅ Approved and optionally add comments.
2
If Rejected, add comments and click Re-Generate — AI will revise the plan based on feedback.
3
Once both approve, select a Jira Project, enter the admin password, and push — Epics, Stories and Subtasks are created in Jira automatically.
💡 Use Export Excel to download the full plan as a workbook regardless of Jira status.
